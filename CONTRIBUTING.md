# Contributing to Library Management System

Thank you for your interest in contributing to the Library Management System (LMS).

## How to Contribute

1. Fork the repository.
2. Clone your fork to your local machine.
3. Create a new branch for your feature or bug fix.
4. Make your changes and test them thoroughly.
5. Commit your changes with a clear commit message.
6. Push your branch to GitHub.
7. Create a Pull Request describing your changes.

## Development Setup

1. Install Python and Django.
2. Clone the project repository.
3. Create a virtual environment.
4. Install dependencies from requirements.txt.
5. Run database migrations.
6. Start the development server.

## Coding Standards

- Follow PEP 8 guidelines.
- Write clean and readable code.
- Add comments where necessary.
- Test all new features before submitting.

## Reporting Issues

If you find a bug or have a feature request, please create an issue with detailed information.

Thank you for helping improve this project!
